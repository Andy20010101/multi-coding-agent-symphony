# Tests

Tests are written before production code for each implementation task.

Initial test suites should cover:

- Contract schemas.
- Artifact Store.
- Session Event Log.
- Failure Taxonomy.
- Policy Engine.
- Capability Registry.
- Context Builder.
- Runtime Adapter conformance.
- Router/Scheduler.
- Verifier.
- Eval Replay Plugin.

The concrete test framework is not chosen yet. Keep test intent tied to behavior scenarios until the first implementation stack is selected.
