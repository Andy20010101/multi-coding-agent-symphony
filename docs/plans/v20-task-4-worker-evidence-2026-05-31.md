## Summary

Task-4 closes the Closeout Gaps panel gap for the active goal Workbench.

Implemented changes:

- `goal-closeout-report.v1` summaries now include `releaseReadySource`.
- Closeout release readiness is true only when task evidence is complete, all release gates are passed, and `release.ready-declared` is present through an explicit `goal-event-log.v1:*` source.
- The Workbench Closeout Gaps projection no longer reads the active ledger to fill closeout readiness fields. It displays `missing`, `releaseGates`, `releaseReady`, and `releaseReadySource` from `goal-closeout-report.v1`.
- Closeout missing release gate rows now surface the report's `gate`, `gateId`, and `status` fields.

The existing task-1 through task-3 Workbench changes and previously generated static assets were left in place. `pnpm workbench:build` regenerated the Workbench static JS asset hash.
