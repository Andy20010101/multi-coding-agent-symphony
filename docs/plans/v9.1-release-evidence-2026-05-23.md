# v9.1 Release Evidence

Date: 2026-05-23
Status: local verification passed; commit, push, CI, and `v9.1` tag pending
Release target: `v9.1`

## Scope

v9.1 ships Workbench diagnostics and evidence polish on top of the v9 local read-only Workbench:

- Run filters for `all`, `passed`, `failed`, `dry-run`, `real`, `scan`, and `verify`.
- Dashboard `runStats` for recent 5 runs, failed count, verifier pass rate, and artifact missing state.
- Risk summaries for dirty git, failed verifier, unsupported requests, external calls, project/runtime writes, missing artifacts, and missing optional tools.
- Run details reorganized into `Intent`, `Route`, `Safety`, `Execution`, `Verification`, `Artifacts`, and `Changes`.
- Artifact previews with clearer missing, malformed JSON, directory truncation, and 200 KiB file truncation metadata.
- Grouped copy-only commands under `Inspect`, `Verify`, `Artifacts`, and `Real-agent gates`.
- Readiness recommendations that point to copy-only inspection or repair commands when tools or state need attention.

The release keeps the v9 safety boundary: the browser UI remains read-only, non-GET requests return `405`, artifacts are previewed only through registered refs, file previews stay capped at 200 KiB, and the Workbench does not execute commands, write files, delete files, retry runs, call models, or read arbitrary paths.
