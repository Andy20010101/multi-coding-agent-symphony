# v9.1 Plan: Workbench Diagnostics and Evidence Polish

Date: 2026-05-23
Status: implemented and locally verified
Baseline: `v9`
Primary CLI: `symphony`

## Summary

v9.1 improves the local read-only `Symphony Workbench` so an operator can quickly judge project health, recent run trust, visible risks, and the next copy-only commands to inspect or continue work.

The v9 safety boundary remains unchanged: the browser UI is read-only, all non-GET requests return `405`, artifact previews only read registered run-state references, file previews stay capped at 200 KiB, and the Workbench does not execute commands, write files, delete files, retry runs, call models, or read arbitrary paths.

## Implemented Changes

- Added run filters for `all`, `passed`, `failed`, `dry-run`, `real`, `scan`, and `verify` in the Workbench UI and as an additive `GET /api/runs?filter=<filter>` query.
- Added snapshot-level `runStats` with recent 5 runs, failed run count, verifier pass rate, artifact missing status, and filter counts.
- Added snapshot, run, and readiness `riskSummary` objects for dirty git, failed verifier, unsupported requests, external calls, project writes, runtime writes, missing artifacts, and missing optional tools.
- Reorganized run details into `Intent`, `Route`, `Safety`, `Execution`, `Verification`, `Artifacts`, and `Changes` sections.
- Improved artifact previews with structured directory `entryCount` and `limit`, malformed JSON parse messages, missing-artifact messaging, and explicit truncation metadata for the 200 KiB preview cap.
- Added grouped copy-only command recommendations under `Inspect`, `Verify`, `Artifacts`, and `Real-agent gates`.
- Made readiness recommendations more specific when pnpm, git, GitHub auth/CI, dirty git state, or optional real CLIs need attention.

## Public Contract Additions

Existing routes remain compatible:

- `GET /`
- `GET /api/health`
- `GET /api/summary`
- `GET /api/readiness`
- `GET /api/runs`
- `GET /api/runs/latest`
- `GET /api/runs/<run-id>`
- `GET /api/runs/<run-id>/timeline`
- `GET /api/runs/<run-id>/artifacts/<kind>`

Additive behavior and fields:

- `GET /api/runs?filter=<filter>` returns the same run list contract with `filter`, `availableFilters`, and filtered `runs`.
- `GET /api/summary` may include `runStats`, `riskSummary`, and `commandGroups`.
- Compact run objects may include `artifactStatus`, `riskSummary`, and `commandGroups`.
- `GET /api/readiness` may include `riskSummary` and `commandGroups`.
- Artifact previews may include `entryCount`, `limit`, `previewLimitBytes`, `message`, and detailed `parseError`.
