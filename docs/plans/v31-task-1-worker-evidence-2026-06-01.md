## User-visible value

Workbench now shows when an active task can enter main verification and when it must first go through review, revision, or adoption recovery. The panel exposes the concrete blocker instead of relying on branch names, file names, prompt text, task titles, command text, or frontend state.

## Explicit State Sources

Readiness uses these backend sources:

- `goal-progress-ledger.v1`: task status, evidence refs, and event-backed reviewer verdict.
- `goal-event-log.v1`: latest reviewer verdict and main-verification events for the active task.
- `goal-next-action.v1`: current task, role, phase, and handoff reason.
- `goal-closeout-report.v1`: closeout missing kinds for the active task.
- `goal-operation-runs.v1`: scoped `adoption-plan` and `adoption-confirm` operations.
- `symphony.console-adoption-inspect`: journal status, latest confirmation run, and worktree hash match fields when a frozen adoption plan exists.

Ignored inference sources:

- Branch names.
- File names.
- Commit messages.
- Prompt text.
- Task titles.
- Copy-only command text.
- Frontend component state.

## Implementation Summary

- Updated `projectMainVerificationReadiness` so `canEnterMainVerification` is derived from explicit reviewer approval, main-verification events, closeout gaps, and adoption operation/inspect state.
- Added adoption-aware blockers: a confirmed `adoption-plan` for the same goal/task blocks main verification until a confirmed `adoption-confirm` run reports `passed`; failed adoption confirmation is shown as a blocker.
- Removed branch/git readiness from the main-verification readiness decision. The panel still shows copy-only terminal guidance for ff-only merge, but the branch text is not used as state.
- Changed the main-verification gate command to use `<main-verification-evidence-ref>` instead of generating a filename-derived evidence ref.
- Added Workbench UI fields for adoption state, blockers, explicit state sources, and ignored inference sources.
- Updated Workbench tests and docs for the v31 readiness model.
